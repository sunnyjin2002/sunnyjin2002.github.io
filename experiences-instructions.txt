Implementation requirements:

1. HTML
Create or update a section like this:

<section class="timeline-section" id="experience">
  <h2>Experience & Education</h2>

  <div class="timeline" id="timeline">
    <div class="timeline-line"></div>
  </div>
</section>

Load the JavaScript file after the timeline HTML.

2. JavaScript
Create a timeline rendering script.

The script should:
- Store timeline entries in the timelineItems array.
- Parse ISO date strings into timestamps.
- Treat null end dates as the current date.
- Compute the timeline bounds using the earliest start date and the latest end date.
- For each item, compute an anchor date.
- Use the midpoint between start and end as the anchor date.
- Convert the anchor date into a top percentage:
  topPercent = ((anchorDate - timelineStart) / (timelineEnd - timelineStart)) * 100
- Sort items chronologically by anchor date.
- Dynamically create article elements for each timeline card.
- Apply classes based on side and type:
  timeline-card
  timeline-card-left or timeline-card-right
  timeline-card-education or timeline-card-work
- Apply the calculated top percentage using inline style:
  card.style.top = `${topPercent}%`
- Inject the generated cards into #timeline.
- Keep the central line as a separate element.

3. CSS
Style the timeline so that on desktop:
- The timeline section is centered and max-width constrained.
- The timeline container has position: relative.
- The center line is absolutely positioned at left: 50%.
- Cards are absolutely positioned.
- Left-side cards are on the left of the center line.
- Right-side cards are on the right of the center line.
- Cards are vertically centered on their calculated position using transform: translateY(-50%).
- Each card has a dot that aligns with the center line.
- Cards have padding, rounded corners, border, subtle shadow, and readable typography.
- Education and work cards can have slight visual distinction, but keep the design clean.

4. Mobile responsiveness
At screen widths below around 768px:
- Collapse the timeline into a single-column layout.
- Do not use absolute positioning for cards on mobile.
- Use normal document flow with flex-column or block layout.
- Keep the vertical line on the left side.
- All cards should appear to the right of the line.
- Sort cards chronologically.
- Override inline top positioning with top: auto !important.
- Ensure the timeline remains readable on mobile.

5. Overlap handling
Start with a simple approach:
- Use a generous desktop min-height, around 1700px to 1900px.
- This allows close dates to have enough space.
- Do not manually hard-code offsets for individual cards.
- If overlapping still occurs, add a generic collision-avoidance helper, but do not solve overlap by manually assigning row numbers.

6. Code quality
- Keep the code simple and readable.
- Use semantic HTML where practical.
- Do not use external dependencies.
- Do not use React.
- Do not use CSS grid row placement for card positions.
- Do not use hard-coded row numbers.
- Make it easy to add a new timeline item by editing only the timelineItems array.
